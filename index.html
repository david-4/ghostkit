Typography

Type is all set with the rems, so font-sizes and spacial relationships can be responsively sized based on a single HTML font-size property. Transparent never changes the font-size but it's there in case you need it for your project. All measurements are still based on 10px though so, an <h1> with 5.0rem font-size just means 50px. single font-size property.

The typography is based on the Raleway font, served by Google Fonts server. If you want to use another local font during to high loading times, you can. Just include the font and add it to the body tag with alternate fonts and a sans-serif fallback.

Classes are also provided for every heading, containing its specific letter-spacing, font-size and line-height. Use the classes .h1 - .h6 to add the specific style to any element.

Lists

Transparent supports different types of lists for every usage and every skin has it's own benefits. Lists have multiple use-cases and can be used for nearly anything. That's why Transparent supports more than the "normal" list types.

Unordered Lists can be used with the class .raw for an unstyled list or with the skin .horizontal which can be used as a navigation or suitable cases.

Ordered Lists support the .vintage style for roman-styles (I, IV, XII and so on) or you can use the .alphabetic skin for an alphabetical enumeration.

Images

Resizing or cropping of images is often used on Website. The most common types are included in this UI Kit. Three different types are included, the .img-rounded, .img-circle and .img-responsive. First, the rounded is used for images with slightly rounded corners. The circle one is used for totally round images and the responsive one can be used to auto-resize the image on shrink/enlarge.

Tipp: use the .img-framed class for a thin 3px border around it.

Media Queries

Transparent uses media queries with the min-width property, this means you have to write the code mobile first. It means all styles outside of a query apply to all devices, then larger devices are targeted for enhancement. This prevents small devices from having to parse tons of unused CSS. There are four sizes:

Name Description Range in px
Desktop HD For all large monitors and huge screens 1200px and more
Desktop General query for desktop computer 750px - 1000px
Tablet Used for normal tablets such as iPad and Galaxy Tab 550px - 750px
Phablet Mixins between Phones and Tablets 400px - 550px
Mobile For mobile phones and smaller devices 0 - 400px
